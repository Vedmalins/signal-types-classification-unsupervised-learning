# Signal Types Classification — Unsupervised Learning

Кластеризация сцинтилляционных сигналов методами машинного обучения без учителя для разделения различных типов физических событий.

## О проекте

Проект выполнен в рамках дисциплины **«Машинное обучение без учителя»**.

Цель работы — исследовать структуру сигналов сцинтилляционного детектора и выполнить их автоматическую кластеризацию на три группы:

* сигналы первого типа частиц;
* сигналы второго типа частиц;
* аномальные или неопределённые сигналы.

Исходные данные предоставлены в рамках соревнования Kaggle **Signal Types Classification** и содержат **23 479 зарегистрированных сигналов**, каждый из которых представлен набором служебных характеристик и временным рядом из 500 отсчётов формы импульса.

В ходе работы были исследованы различные способы представления сигналов, выполнено извлечение физических признаков, проведено сравнение нескольких алгоритмов кластеризации и сформировано итоговое решение для соревнования Kaggle.

## Постановка задачи

Необходимо решить задачу кластеризации без использования размеченных данных.

Требуется разделить сигналы на три кластера:

* **Кластер 0** — сигналы первого типа частиц;
* **Кластер 1** — сигналы второго типа частиц;
* **Кластер 2** — аномальные или смешанные сигналы.

Для решения использовались методы обучения без учителя, а также анализ формы физических сигналов.

## Структура исследования

### 1. Исследовательский анализ данных (EDA)

Выполнен анализ структуры датасета:

* анализ размерности данных;
* поиск пропусков и дубликатов;
* исследование статистических характеристик признаков;
* визуализация случайных сигналов;
* корреляционный анализ признаков;
* поиск закономерностей и неинформативных признаков.

---

### 2. Предобработка данных

Выполнены следующие этапы:

* выделение служебных признаков и временных рядов;
* удаление константных признаков;
* масштабирование признаков методом RobustScaler;
* подготовка данных к кластеризации.

---

### 3. Feature Engineering

Из формы импульса были извлечены физически интерпретируемые характеристики:

* baseline (базовый уровень сигнала);
* минимальное значение сигнала;
* максимальная амплитуда импульса;
* положение максимума;
* площадь под импульсом;
* энергия сигнала;
* ширина импульса;
* время восстановления;
* статистические характеристики формы сигнала;
* асимметрия и эксцесс.

---

### 4. Снижение размерности

Для анализа структуры данных использовался метод главных компонент (**PCA**).

Основные результаты:

* первые две главные компоненты объясняют более 87% дисперсии инженерных признаков;
* первые пять компонент объясняют более 96% дисперсии;
* PCA позволил визуализировать структуру кластеров и уменьшить размерность пространства признаков.

---

### 5. Кластеризация

Были исследованы следующие алгоритмы:

* K-Means;
* Gaussian Mixture Model (GMM);
* Agglomerative Clustering;
* DBSCAN.

Для оценки качества использовались:

* Silhouette Score;
* Davies–Bouldin Index;
* Calinski–Harabasz Index.

---

### 6. Сравнение моделей

| Модель                   | Silhouette Score |
| ------------------------ | ---------------: |
| K-Means (k=3)            |            0.569 |
| GMM tied covariance      |            0.562 |
| Agglomerative Clustering |            0.453 |
| DBSCAN                   |           0.611* |

* DBSCAN выделил дополнительный шумовой кластер и поэтому не рассматривался как итоговое решение.

## История экспериментов Kaggle

В ходе работы были протестированы различные способы представления сигналов и алгоритмы кластеризации.

| Подход                                   | Основная стратегия          | Public Score |
| ---------------------------------------- | --------------------------- | -----------: |
| GMM tied на PCA полной формы сигнала     | Полная форма сигнала + PCA  |  **0.51697** |
| KMeans на PCA полной формы сигнала       | Полная форма сигнала + PCA  |  **0.50487** |
| GMM tied на baseline-corrected pulse PCA | Коррекция baseline + PCA    |  **0.50129** |
| GMM на окне импульса 140:200             | Локальное окно импульса     |  **0.41543** |
| GMM tied на инженерных признаках         | Feature Engineering         |  **0.41198** |
| KMeans на инженерных признаках           | Feature Engineering         |  **0.40078** |
| GMM по PSD tail/total                    | Физические PSD-признаки     |  **0.33106** |
| Agglomerative на инженерных признаках    | Иерархическая кластеризация |  **0.32488** |

## Лучшее решение

Наиболее качественный результат показала модель:

**Gaussian Mixture Model (GMM) с общей ковариационной матрицей (tied covariance)** на PCA-признаках полной формы сигнала.

Лучший результат на Kaggle:

**Public Score = 0.51697**

Полученные результаты показали, что наиболее информативной является полная временная форма сигнала, а не только набор инженерных признаков.

## Основные выводы

В ходе исследования были выявлены следующие закономерности:

* датасет не содержит пропусков и дубликатов;
* часть признаков имеет низкую информативность;
* форма импульса содержит больше информации, чем агрегированные характеристики;

* использование полной формы сигнала даёт более качественную кластеризацию по сравнению с инженерными признаками.
